## Delivery Notes

- **"Communications blackout"** — powerful framing that resonates with telecom engineers
- **Pause** after "real-world consequences" — this differentiates you from textbook security
- **Emphasize** IoT and firmware — these are areas most security candidates don't mention
- **On kood/Jõhvi** — shows continuous learning, which Ericsson values
- **Final sentence** — "connections can never be used against people" is a strong closer

## Key Topics to Have Ready

- 5G network security — radio access network (RAN) hardening
- IoT device security and firmware integrity verification
- Smart industry solutions — IIoT security in manufacturing
- Network protocol security — MQTT, CoAP, cellular protocols
- Supply chain security for telecom infrastructure
- How Ericsson Tallinn contributes to global platform engineering
