"You're building the next generation of Estonian developers. I want to make sure security is part of that DNA from day one — not something bolted on after graduation."

"I'm Tom Kristian Abel — I'm one of your students. I'm also a security researcher, founder of ProksiAbel OÜ, and CTO at MatX.ee. My team won FELLIN HÄKK 2026. I bring a perspective that most students don't have — I've already built and broken production systems."

"//kood is unique because you're not just teaching code — you're transforming careers. I'm living that transformation right now. **[WHAT I BRING — 15s]**
"I bring real-world security expertise that I want to share. I've done penetration testing, reverse engineering, vulnerability research, and built offensive tooling in Golang. My pet project — a production-grade async API with FastAPI, Docker, Redis, and PostgreSQL — demonstrates the kind of engineering rigor that goes beyond tutorials. I can contribute to the community, mentor students on security, and help shape how security is taught."

**[CLOSING — 5s]**
"//kood gives people a second chance at a tech career. I want to make sure that second chance includes the skills to build software that's not just functional, but secure."

## Delivery Notes

- **"I'm one of your students"** — immediately creates a different dynamic, shows loyalty
- **Pause** after "bolted on after graduation" — this is a constructive criticism wrapped in a compliment
- **Martin Villig reference** — shows you know the leadership, done your research
- **Emphasize** "contribute to the community" — //kood is community-driven, this resonates
- **Final sentence** — "not just functional, but secure" is a concise value proposition

## Key Topics to Have Ready

- Security curriculum integration — what security topics should //kood teach?
- Mentoring opportunities — how can experienced students contribute?
- Martin Villig's vision for //kood and how security fits
- Tehnopol ecosystem connections and how //kood graduates plug in
- Product mentoring and code review — how security perspectives improve both
